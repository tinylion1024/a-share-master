# A-Shares Master 架构设计

## 设计目标

1. **模块化**：各功能模块独立，可单独升级和维护
2. **可扩展**：支持插件机制，可扩展数据源、指标、报告模板
3. **可组合**：模块之间可按需组合成分析流水线
4. **可测试**：核心逻辑与外部依赖分离，便于单元测试

## 核心模块

### 四维分析引擎 (Four-Dimensional Analyzer)

| 维度 | 说明 |
|------|------|
| A. 新闻与政策 | MX Search 宏观与行业政策 |
| B. 情绪与资金 | TGB 社区情绪 + MX 主力资金 |
| C. 市场模式与流动性 | 量价模式、均线支撑 |
| D. 风险与合规 | check_risk.py 强制扫描 |

### 风险扫描器 (Risk Checker)

- R3 红线检测（立案调查、退市风险、业绩盲盒）
- 流动性风险
- 仓位风险
- 相关性风险

### 评级系统 (Rating System)

- Risk: R1/R2/R3
- Opportunity: O1/O2/O3
- Certainty: C1/C2/C3
- Comfort: ⭐(1-5)

## 插件系统

### 插件类型

| 插件类型 | 接口 | 说明 |
|----------|------|------|
| 数据源插件 | `DataSourcePlugin` | 获取市场数据 |
| 情绪源插件 | `SentimentPlugin` | 获取社区情绪 |
| 指标插件 | `IndicatorPlugin` | 计算技术指标 |

### 内置插件

| 插件名 | 类型 | 说明 |
|--------|------|------|
| mx_data | 数据源 | MX API 数据源 |
| akshare | 数据源 | akshare 数据源（备用） |
| taoguba | 情绪源 | 淘股吧情绪 |

## 扩展指南

### 添加新模块

1. 在 `scripts/` 下创建新脚本
2. 遵循现有脚本的接口规范
3. 在 SKILL.md 中注册入口

### 添加新插件

1. 实现插件接口
2. 在插件注册表注册
3. 配置默认使用
