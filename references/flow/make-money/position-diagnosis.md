# 持仓诊断

> 分析持仓股是否该持有/加/减/清

## 触发词

- "我的XXX要不要卖"
- "持仓XXX怎么办"
- "XXX还能持有吗"
- "帮我看看持仓"

## 前置确认

- [ ] 确认股票代码
- [ ] 确认买入成本价
- [ ] 确认当前持仓仓位
- [ ] 确认持仓时长

## 分析步骤

### Step 1：持仓状况评估

| 持仓状态 | 盈亏 | 操作倾向 |
|----------|------|----------|
| 盈利 | +X% | 持有/分批止盈 |
| 微盈 | 0~+3% | 谨慎持有 |
| 微亏 | -3%~0 | 观察是否破止损 |
| 亏损 | >-7% | 是否破止损 |
| 深亏 | <-7% | 止损/解套方案 |

### Step 2：四维复评

| 维度 | 重新评估 |
|------|----------|
| 机会⭐ | 逻辑是否还在 |
| 安全⭐ | 是否触发R3 |
| 确定⭐ | 上涨逻辑是否破坏 |
| 舒适⭐ | 持有心态 |

### Step 3：决策树

```
亏损>-7% → 止损
亏损<-15% → 解套方案
盈利且逻辑在 → 持有/加
盈利但逻辑破 → 减/清
```

## 输出规范

```markdown
## [持仓诊断]
股票：XXX 代码：XXX
成本价：XX.X  现价：XX.X  盈亏：+X%/-X%

**诊断结论**：持有 / 加仓 / 减仓 / 清仓

| 操作 | 价位 | 理由 |
|------|------|------|
| 持有 | - | 逻辑未破 |
| 止损价 | XX | -7%红线 |
| 目标价 | XX | 分批止盈 |

**核心逻辑是否还在**：是/否
**后续关注点**：XXX
```

## 硬规则检查

- ❌ 亏损>-7% → 必须止损
- ❌ R3触发 → 卖出
- ❌ T+1当日 → 注意不能卖
