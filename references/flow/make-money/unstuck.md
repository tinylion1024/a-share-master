# 解套

> 被套后的应对方案

## 触发词

- "被套了怎么办"
- "XXX亏了XX%"
- "怎么解套"
- "要不要补仓"

## 前置确认

- [ ] 确认股票代码
- [ ] 确认买入成本价
- [ ] 确认当前亏损幅度
- [ ] 确认可用资金

## 解套方案决策树

```
亏损幅度     方案
<7%       → 止损出局（不拖延）
7%-15%    → 止损/等反弹减亏
15%-30%   → 躺平/补仓解套
30%-50%   → 躺平/换股
>50%      → 割肉/等回本（不推荐）
```

## 分析步骤

### Step 1：套牢程度评估

| 程度 | 亏损幅度 | 方案 |
|------|----------|------|
| 轻度 | <7% | 止损 |
| 中度 | 7-15% | 等反弹减亏/止损 |
| 重度 | 15-30% | 补仓/躺平 |
| 深套 | 30-50% | 躺平/换股 |
| 严重 | >50% | 割肉/长等 |

### Step 2：补仓条件判断

满足以下条件可考虑补仓：

- [ ] 亏损>15%
- [ ] 股票R3安全⭐≥3
- [ ] 逻辑未破坏（基本面未变）
- [ ] 有足够资金（补仓后不超仓位上限）
- [ ] 大盘环境不是单边下跌

### Step 3：补仓策略

| 方案 | 条件 | 操作 |
|------|------|------|
| 金字塔补仓 | 分批越跌越买 | 20%/30%/50% |
| 等额补仓 | 每跌X%补同等金额 | 成本摊平 |
| 不补仓 | 大熊市/逻辑破 | 躺平/割肉 |

## 输出规范

```markdown
## [解套方案]
股票：XXX 代码：XXX
成本价：XX.X  现价：XX.X  亏损：-X%

**解套方案**：止损 / 补仓 / 躺平 / 换股

补仓计划（如选择补仓）：
| 次数 | 价位 | 补仓比例 | 累计仓位 |
|------|------|----------|----------|
| 第一次 | XX | X成 | X成 |
| 第二次 | XX | X成 | X成 |

**时间预期**：X-X个月
**解套概率**：XX%
**风险提示**：XXX
```

## 硬规则检查

- ❌ 补仓后单只>30%仓位 → 禁止
- ❌ 总仓位>50% → 禁止补仓
- ❌ 逻辑已破 → 不补仓
